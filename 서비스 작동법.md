# 서비스 작동법

이 문서는 `buzzy pages` 서비스를 로컬 개발 환경과 EC2 운영 환경에서 실행하는 방법을 정리한 문서입니다.

## 1. 사전 준비

필요한 실행 환경은 다음과 같습니다.

- Node.js 18 이상
- npm 9 이상
- MySQL 8.0 이상
- 운영 서버에서는 PM2와 Nginx

처음 내려받은 환경에서는 의존성을 설치합니다.

```bash
# 서버 의존성
cd server
npm install

# 클라이언트 의존성
cd ../client
npm install
```

## 2. 환경 변수 설정

서버 환경 변수 파일을 생성합니다.

```bash
cd server
cp .env.example .env
```

`server/.env`에서 아래 값을 실제 환경에 맞게 설정합니다.

```env
NODE_ENV=development
PORT=3000
DATABASE_URL="mysql://bookuser:your_password@localhost:3306/book_discussion"
CORS_ORIGIN=
FILE_STORAGE_DRIVER=local
JWT_SECRET=your_jwt_secret_here
JWT_REFRESH_SECRET=your_jwt_refresh_secret_here
KAKAO_API_KEY=your_kakao_api_key
NAVER_CLIENT_ID=your_naver_client_id
NAVER_CLIENT_SECRET=your_naver_client_secret
GEMINI_API_KEY=your-gemini-api-key
GEMINI_MODEL=gemini-2.5-flash
```

클라이언트 환경 변수 파일도 생성합니다.

```bash
cd ../client
cp .env.example .env
```

로컬 개발에서는 `client/.env`의 `VITE_API_BASE_URL`을 비워두면 Vite proxy 기준으로 `/api`를 사용합니다.

```env
VITE_API_BASE_URL=
```

정적 호스팅이나 별도 API 도메인을 사용할 때만 API 주소를 지정합니다.

```env
VITE_API_BASE_URL=https://api.example.com/api
```

## 3. 데이터베이스 준비

서버 디렉터리에서 Prisma 마이그레이션을 적용합니다.

```bash
cd server
npx prisma migrate dev
npx prisma generate
```

운영 환경에서 배포용으로 마이그레이션만 적용할 때는 다음 명령을 사용합니다.

```bash
cd server
npx prisma migrate deploy
```

## 4. 로컬 개발 실행

개발 중에는 서버와 클라이언트를 각각 실행합니다.

터미널 1: 백엔드 서버 실행

```bash
cd server
npm run dev
```

백엔드는 기본적으로 `http://localhost:3000`에서 실행됩니다.

터미널 2: 프론트엔드 개발 서버 실행

```bash
cd client
npm run dev
```

프론트엔드는 기본적으로 `http://localhost:5173`에서 실행됩니다.

상태 확인:

```bash
curl http://localhost:3000/api/health
```

정상 응답 예시:

```json
{"status":"ok"}
```

## 5. 운영 빌드

운영 환경에서는 클라이언트와 서버를 모두 빌드합니다.

```bash
# 루트 기준
cd client
npm run build

cd ../server
npm run build
```

서버는 프로덕션 모드에서 `client/dist`를 정적 파일로 함께 제공합니다. 따라서 PM2는 백엔드 서버만 실행하면 됩니다.

## 6. PM2 설정 및 서비스 실행

PM2 설정 파일은 루트의 `ecosystem.config.js`를 사용합니다.

현재 설정:

```js
module.exports = {
  apps: [{
    name: 'book-discussion',
    cwd: 'server',
    script: 'dist/index.js',
    env: {
      NODE_ENV: 'production',
      PORT: 3000,
    },
    instances: 1,
    autorestart: true,
    watch: false,
    max_memory_restart: '500M',
  }],
};
```

운영 서버에서 서비스 시작:

```bash
# 루트 기준
pm2 start ecosystem.config.js
```

상태 확인:

```bash
pm2 status
```

로그 확인:

```bash
pm2 logs book-discussion
```

재시작:

```bash
pm2 restart book-discussion
```

중지:

```bash
pm2 stop book-discussion
```

삭제:

```bash
pm2 delete book-discussion
```

서버 재부팅 후에도 PM2 프로세스가 살아나도록 저장합니다.

```bash
pm2 save
pm2 startup
```

`pm2 startup` 실행 후 출력되는 `sudo env ... pm2 startup ...` 명령을 한 번 더 실행해야 자동 시작 설정이 완료됩니다.

## 7. 배포 순서 요약

EC2에서 코드를 갱신한 뒤에는 아래 순서로 배포합니다.

```bash
# 1. 최신 코드 반영
git pull

# 2. 서버 의존성 및 DB 반영
cd server
npm install
npx prisma migrate deploy
npm run build

# 3. 클라이언트 빌드
cd ../client
npm install
npm run build

# 4. PM2 재시작
cd ..
pm2 restart book-discussion
pm2 status
```

처음 배포하는 서버라면 마지막 단계에서 `pm2 restart` 대신 다음 명령을 사용합니다.

```bash
pm2 start ecosystem.config.js
pm2 save
```

## 8. Nginx 연결

Nginx 설정 예시는 `nginx/book-discussion.conf`에 있습니다.

핵심 설정은 외부 80 포트 요청을 PM2가 실행 중인 Express 서버의 `127.0.0.1:3000`으로 전달하는 것입니다.

```nginx
location / {
    proxy_pass http://127.0.0.1:3000;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

설정 적용 후 확인:

```bash
sudo nginx -t
sudo systemctl reload nginx
```

## 9. 자주 쓰는 명령어

```bash
# 개발 서버
cd server && npm run dev
cd client && npm run dev

# 운영 빌드
cd client && npm run build
cd server && npm run build

# PM2
pm2 start ecosystem.config.js
pm2 restart book-discussion
pm2 stop book-discussion
pm2 logs book-discussion
pm2 status

# 헬스 체크
curl http://localhost:3000/api/health
```
