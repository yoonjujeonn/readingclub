# 캡스톤 발표 PPT 제작 가이드

## 발표의 핵심 목표

> "왜 이 문제가 중요하고, 왜 우리 서비스가 필요하며, 왜 이런 방식으로 해결했는가"

를 평가자에게 명확하게 전달하는 것.

---

# 전체 PPT 구성 (10장)

## 1. 표지

### 목표                                                                                                     
서비스가 무엇인지 한 번에 이해시키기

### 포함 요소
- 서비스명
- 한 줄 소개: "함께 읽고, 함께 기록하고, 함께 토론하는 AI 독서 모임 플랫폼"
- 메인 화면 Mockup
- 팀명 / 팀원

---

## 2. 문제 정의 + 기존 방식의 한계 (평가: 문제 정의 15점)

### 목표
왜 이 서비스가 필요한지 공감시키고, 기존 방식으로 해결 안 되는 이유 설명

### 구성
- 핵심 질문: "어떻게 하면 온라인 독서모임에서도 깊이 있는 토론과 지속적인 참여를 만들 수 있을까?"
- 사용자 Pain Point (2~3개 핵심만)
  - 참여도 차이 / 토론거리 부족 / 기록 휘발 / 스포일러
- 기존 도구(카톡, 노션, 독서앱)의 한계를 한 줄씩 대비

### 디자인 방향
- 좌: Pain Point 키워드 / 우: 기존 도구 한계
- 큰 텍스트, 여백 활용

---

## 3. 서비스 소개 + 사용자 흐름

### 목표
서비스 핵심 구조와 사용 흐름을 한 장에 전달

### 포함 요소
- 서비스 한 줄 설명
- 핵심 기능 요약 (메모 / 진도 기반 공개 / 스레드 토론 / AI 인사이트)
- 사용자 흐름도: 책 선택 → 모임 참여 → 메모 작성 → 진도 기반 열람 → AI 추천 → 토론

### 디자인 방향
- 상단: 기능 아이콘 요약
- 하단: 흐름도 (화살표 연결)

---

## 4. 핵심 기능 — 진도 기반 메모 공개

### 목표
서비스 차별점 강조

### 핵심 가치
- 스포일러 방지
- 같은 진도 사용자 간 공감 형성
- 몰입 유지

### 중요
"왜 필요한 기능인가"를 중심으로 설명

---

## 5. 핵심 기능 — AI 인사이트 + 스레드 토론

### 목표
AI 기능과 토론 구조가 실제 문제를 어떻게 해결하는지 설명

### AI 인사이트
- 키워드 요약 / 감정 분석 / 토론 주제 추천
- ❌ "Bedrock 사용" → ⭕ "토론거리 부족 문제 해결"

### 스레드 토론
- 주제별 스레드 / 종료 기간 설정 / 인사이트 자동 저장
- "흩어지는 대화" → "축적되는 토론"

### 디자인 방향
- 좌: AI 인사이트 화면 / 우: 스레드 토론 화면
- 각각 해결하는 문제를 한 줄로 명시

---

## 6. 기술 설계 (평가: 기술 설계 25점)

### 목표
왜 이 기술을 선택했는지, 의사결정 흐름 설명

### 구성 방식
기술 나열 ❌ → 문제-기술 연결 ⭕

| 해결 문제 | 기술 선택 | 선택 근거 |
|---|---|---|
| AI 요약 및 토론 추천 | Amazon Bedrock (Claude) | 한국어 성능, AWS 통합 |
| 데이터 저장 | AWS RDS (PostgreSQL) | 관계형 데이터, Prisma 연동 |
| 프론트엔드 | React + Vite | 빠른 개발, SPA 적합 |
| 백엔드 API | Express + TypeScript | 타입 안정성, 빠른 프로토타이핑 |
| 배포 | AWS EC2 + Nginx | 비용 효율, 풀스택 단일 서버 |

### 아키텍처 다이어그램
- Client → Nginx → Express API → RDS / Bedrock
- 간결한 블록 다이어그램

---

## 7. 구현 완성도 + 데모 (평가: 구현 완성도 30점)

### 목표
실제 동작하는 서비스임을 증명

### 포함 요소
- 구현 완료 기능 체크리스트
- 배포 URL (접근 가능 상태)
- 라이브 데모 또는 녹화 영상
- 핵심 시나리오 시연:
  1. 회원가입/로그인
  2. 모임 생성 + 책 선택
  3. 메모 작성 → 진도 기반 공개 확인
  4. AI 인사이트 생성
  5. 스레드 토론 참여

### 중요
- "미래 계획"이 아닌 "현재 동작하는 것" 중심
- 안정성/성능 언급 (에러 핸들링, 로딩 상태 등)

---

## 8. 실용성 + 기대 효과 (평가: 실용성/확장 15점)

### 목표
실제 활용 가능성과 확장 가능성 전달

### 실용성
- 실제 독서 모임에서 사용 가능한 수준
- 모바일 반응형 지원

### 확장 가능성
- 다른 콘텐츠(영화, 강의) 모임으로 확장 가능
- AI 모델 교체 용이한 구조 (Provider 패턴)

### 기대 효과
- 독서 참여 지속성 증가
- 깊이 있는 토론 가능
- 기록 축적을 통한 독서 습관 형성

---

## 9. 향후 발전 방향

### 주의
2~3개만 간결하게

### 예시
- AI 추천 고도화 (개인화)
- 독서 통계/리포트 기능
- 오프라인 모임 연동

---

## 10. 마무리

### 핵심 메시지
> "독서 기록을 넘어, 함께 읽는 경험을 만드는 플랫폼"

### 포함 요소
- 서비스 핵심 가치 한 줄 재강조
- Q&A 안내

---

# 디자인 가이드

## 유지할 것
- Mockup / 실제 화면 중심
- 여백 활용, 큰 텍스트
- 직관적 구조
- 사용자 가치 중심 설명

## 지양할 것
- 과한 애니메이션 / 전환 효과
- 긴 문장, 텍스트 과다
- 기술 용어 나열

# 발표 가이드

## 시간 배분 (10분 기준)

| 슬라이드 | 시간 |
|---|---|
| 1. 표지 | 30초 |
| 2. 문제 정의 | 1분 |
| 3. 서비스 소개 | 1분 |
| 4. 진도 기반 메모 | 1분 |
| 5. AI + 토론 | 1분 30초 |
| 6. 기술 설계 | 1분 30초 |
| 7. 데모 | 2분 |
| 8. 실용성/기대효과 | 30초 |
| 9. 향후 방향 | 20초 |
| 10. 마무리 | 10초 |

## 예상 질문 준비
- 왜 이 문제가 중요한가?
- 기존 서비스와 차이는?
- AI 환각(hallucination) 문제는 어떻게 대응하나?
- 사람들이 실제 사용할까?
- 일정 내 구현 가능했나?
- 역할 분담은 어떻게 했나?

## 발표 태도
- 자신감 있게, 핵심만 짧고 명확하게
- 질문에 감사 표현 먼저
- 모르는 부분은 솔직하게 인정
