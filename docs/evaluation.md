# 평가 기준
문제 정의의 타당성 (15점)
실제 사용자·현업의 Pain Point를 구체적으로 정의했는지, 기존 방식의 한계와 해결 필요성을 명확히 설명했는지

기술 설계 및 활용 (25점)
문제 해결에 적합한 기술을 선택·조합했는지, 아키텍처 설계의 근거와 의사결정 흐름이 논리적인지

구현 완성도 및 배포 (30점)
실제 동작하는 프로토타입을 완성했는지, 접근·사용 가능한 환경에 배포되었는지, 기본적인 안정성·성능을 갖추었는지

실용성·확장 가능성 (15점)
실제 현업·사용자에게 활용 가능한 수준인지, 다른 도메인으로 확장 가능한 구조인지

발표 구성 및 전달력 (15점)
문제→해결→데모→기대효과 흐름이 논리적인지, 제한 시간 내에 핵심을 명확히 전달했는지
